# 🚀 Деплой Casino Platinum Stars на FirstVDS

## Структура проекта
```
casino-app-platinum-stars-amvera-2/
├── app/                  ← Next.js приложение
├── ASTROBOT/             ← Python Telegram бот
├── prisma/               ← База данных схемы
├── Dockerfile            ← ← нужно скопировать из этой папки
├── docker-compose.firstvds.yml  ← основной compose
├── .env.production       ← переменные для Next.js
└── ASTROBOT/.env.production     ← переменные для бота
```

---

## ШАГ 1 — Купить VPS на FirstVDS

- Тариф: минимум **2 CPU / 4GB RAM / 30GB SSD**
- ОС: **Ubuntu 22.04**
- Получить IP сервера

---

## ШАГ 2 — Подготовить домен

1. Купить домен (reg.ru, nic.ru)
2. В DNS настройках домена добавить **A-запись**:
   ```
   @ → ВАШ_IP_СЕРВЕРА
   ```
3. Подождать 5-30 минут пока DNS обновится

---

## ШАГ 3 — Подключиться к серверу

```bash
ssh root@ВАШ_IP
```

---

## ШАГ 4 — Установить Docker

```bash
apt update && apt upgrade -y
apt install -y curl git unzip nginx certbot python3-certbot-nginx

# Docker
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# Docker Compose plugin
apt install -y docker-compose-plugin
docker compose version  # проверка
```

---

## ШАГ 5 — Загрузить проект на сервер

**С вашего компьютера** (в PowerShell или терминале):
```bash
scp casino-app-platinum-stars-amvera-2.zip root@ВАШ_IP:/root/
```

**На сервере:**
```bash
cd /root
unzip casino-app-platinum-stars-amvera-2.zip
mv casino-app-platinum-stars-amvera-2 casino
cd casino
```

---

## ШАГ 6 — Скопировать файлы деплоя в проект

Загрузите файлы из этой папки (deploy/) на сервер так же через scp,
затем скопируйте в нужные места:

```bash
# Dockerfile → корень проекта
cp /root/Dockerfile /root/casino/Dockerfile

# docker-compose для FirstVDS
cp /root/docker-compose.firstvds.yml /root/casino/docker-compose.firstvds.yml

# .env для Next.js
cp /root/.env.production /root/casino/.env.production

# .env для бота
cp /root/ASTROBOT.env.production /root/casino/ASTROBOT/.env.production
```

---

## ШАГ 7 — Заполнить .env файлы

```bash
# Заменить ВАШ_ДОМЕН на реальный домен в обоих файлах
nano /root/casino/.env.production
nano /root/casino/ASTROBOT/.env.production
```

Найти строки с `ВАШ_ДОМЕН` и заменить на реальный домен, например `mysite.ru`:
```
TELEGRAM_MINI_APP_URL=https://mysite.ru
NEXT_PUBLIC_MINI_APP_URL=https://mysite.ru
BACKEND_BASE_URL=https://mysite.ru
```

---

## ШАГ 8 — Настроить Nginx

```bash
nano /etc/nginx/sites-available/casino
```

Вставить содержимое файла `nginx.conf` (заменив ВАШ_ДОМЕН).

```bash
ln -s /etc/nginx/sites-available/casino /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

---

## ШАГ 9 — Получить SSL сертификат

```bash
certbot --nginx -d ВАШ_ДОМЕН
```

Следовать инструкциям, выбрать автоматический редирект.

---

## ШАГ 10 — Запустить проект через Docker

```bash
cd /root/casino
docker compose -f docker-compose.firstvds.yml up -d --build
```

Дождаться сборки (3-5 минут). Проверить статус:
```bash
docker compose -f docker-compose.firstvds.yml ps
docker compose -f docker-compose.firstvds.yml logs web --tail=50
docker compose -f docker-compose.firstvds.yml logs bot --tail=50
```

## Полезные команды

```bash
# Перезапустить всё
docker compose -f docker-compose.firstvds.yml restart

# Посмотреть логи Next.js
docker compose -f docker-compose.firstvds.yml logs web -f

# Посмотреть логи бота
docker compose -f docker-compose.firstvds.yml logs bot -f

# Остановить всё
docker compose -f docker-compose.firstvds.yml down

# Обновить проект после изменений
docker compose -f docker-compose.firstvds.yml up -d --build

# Зайти в базу данных
docker exec -it casino_postgres psql -U casino -d casino
```

## Возможные проблемы

| Проблема | Решение |
|---|---|
| `Cannot connect to database` | Подождать 30 сек, postgres стартует медленнее |
| `Webhook not set` | Проверить что домен открывается по HTTPS |
| Чеки не приходят в канал | Бот должен быть администратором канала |
| `Port 3000 already in use` | `docker compose down` и запустить снова |
